## Principes de développement

1. **Robustesse avant tout** — le code doit fonctionner durablement
2. **Lisibilité** — quelqu'un d'autre doit pouvoir le lire
3. **Documentation** — chaque script a un header expliquant quoi, pourquoi, comment
4. **Tests** — valider avant de déployer
5. **Versionner** — git commit pour chaque étape significative
6. **Pas de bricolage** — si c'est utile aujourd'hui, le construire pour que ça le soit encore demain
